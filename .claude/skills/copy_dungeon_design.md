# 副本任务设计 Skill

## 适用场景

在本项目里新增或复刻一条副本任务链时使用。重点是把副本拆成可复用的数据和状态，而不是把剧情写死在页面里。

## 设计原则

- 副本内容必须与普通地图资产分离，地图、NPC、怪物都要带副本标记。
- 副本流程必须数据驱动，剧情、阶段、目标、奖励都放进独立 JSON。
- 玩家状态只复用现有的 JSON 型字段，优先放到 `activity_data`，不要为了副本专门加数据库表。
- 战斗结算只负责“打死了什么”，副本进度更新交给独立服务。
- 页面层只负责展示当前阶段、接受任务、完成阶段、领取奖励。
- 副本真实节奏应拆成“任务详情页 -> 回 NPC 交任务页 -> 完成任务确认页 -> 场景领奖页”，不要把所有动作堆在一个页面里。

## 落地步骤

1. 先确认副本入口、NPC、怪物、地图是否都需要副本专属标记。
2. 新建 `data/copy_dungeons.json`，把剧情、阶段、目标、奖励写成结构化数据。
3. 新建副本专属地图和副本专属怪物数据，避免复用普通地图命名。
4. 给玩家副本状态设计一个 JSON 结构，放在 `activity_data.copy_dungeons` 下。
5. 战斗结算里只记录击杀，再调用副本服务更新阶段进度。
6. NPC 页面展示当前阶段和按钮：接受任务、任务详情、交任务。
7. 阶段完成后先进入完成任务确认页，再继续下一阶段。
8. 最终阶段完成后先回副本场景，再单独露出“领取通关奖励”入口。
9. 通关奖励领取完毕后，副本状态必须彻底重置，支持再次进入。

## 推荐状态结构

```json
{
  "copy_dungeons": {
    "huangjin_trial": {
      "accepted": true,
      "stage_index": 0,
      "progress": 3,
      "ready_to_complete": false,
      "completed": false,
      "reward_claimed": false
    }
  }
}
```

## 阶段设计

- 每个阶段只做一件事，目标尽量明确。
- 小怪阶段用数量目标，精英阶段用击败单体目标，最终阶段可以是多首领分目标。
- 阶段完成后要有一个“回到 NPC 再点一次”的确认环节，避免战斗结果直接跳转导致状态不清晰。
- 如果最终阶段有多个首领，不要用一个总数糊过去，任务详情页应逐个显示首领进度。
- 如果存在木筏、钥匙、机关等前置物品，前置物品应作为通路限制，不一定计入任务进度数字。

## 数据字段建议

- 地图：`is_copy_map`、`copy_dungeon_id`
- NPC：`is_copy`、`copy_dungeon_id`、`copy_role`
- 怪物：`is_copy`、`copy_dungeon_id`、`copy_stage`
- 副本定义：`entry_location`、`entry_npc_id`、`story_intro`、`story_outro`、`steps`、`reward`

补充建议：
- 阶段目标页文案用 `objective`
- 阶段接受页剧情文案用 `story`
- 多目标最终阶段建议单独存 `target_monsters` 列表，不要把前置怪也塞进完成计数
- 如果最终通关奖励需要二段领取，保留副本总奖励 `reward`，并在场景页提供单独领取入口

## 检查清单

- 副本入口能把玩家送到副本地图。
- NPC 页面能看到当前阶段、目标、进度。
- 击杀目标怪后会推进阶段进度。
- 完成最后阶段后会进入通关结算页。
- 领取奖励后玩家回到普通场景。
- 所有副本资产都带有明确标识，和普通资产不会混淆。

## 黄巾起义副本已验证流程

- `副·大破黄巾一`：豫州，5 个黄巾士兵

### 基本信息
- 入口 NPC：郭嘉【副本】(广场-北平中区)
- 道具消耗：神游果 x1
- 剧情入口："助郭嘉计定辽东斩二袁平匈奴"
- 副本地图链：易州城(副本) → 神医坊(西) → 须弥山(南) + 北沙漠(北)；易州城 → 卢龙口(东) → 百檀(东) → 白狼山口(东) → 白狼山前(东) → 柳城(北) → 太守府

### 任务类型总结
- 击杀型(6): 1(冒牌神医), 6(敌军探子), 7(精英他顿), 8(辽东兵), 9(精英袁尚), 10(公孙瓒)
- 收集型(2): 2(天香草), 3(蝎毒) — 怪物掉落任务物品，自动拾取
- 递送型(2): 4(秘药→郭嘉), 5(信封→曹操) — 前一阶段奖励链

### 关键机制
- 长链条副本(8+阶段)，混合击杀/收集/递送三种任务类型
- 收集型任务：怪物掉落任务物品(天香草、蝎毒)，自动拾取，完成任务时自动消耗
- 递送型任务链：秘药→信封，前一阶段奖励物品，下一阶段直接交付
- 递送阶段在同一场景原地完成，无需切换地图
- 完成结算页格式：`失去[物品]N个`(收集/递送型) → `获得 物品(绑) x1`(如有) → `『完成任务-XXX』` → 经验+xxx → 银两+xxx → NPC剧情对话 → 提示下一目标 → `[继续]`
- 阶段6、7、8奖励不同(6000/600 vs 3000/300)
- 精英怪带【精】前缀，HP显著高于普通怪(4800 vs 800)
- **副本精英怪是个人精英怪，不是世界BOSS**：只能自己打，血量不共享，自己死亡后重新打时之前打的血量已扣减（保存在 encounter 中）。世界BOSS规则只适用于副本外的精英怪（多人一起打，谁先杀死算谁的）
- **副本内禁止传送**：VIP传送、普通传送、回城、神行在副本内全部禁用，只能通过"放弃副本"离开
  - 实现位置：`blueprints/map_route.py`（teleport/town/shenxing_go）+ `blueprints/vip.py`（do_teleport）
  - 判断条件：`DataService.get_location(player.current_location).get('is_copy_map')`
  - 玩家离开副本场景后，`game.py scene()` 自动清除副本状态（安全网）
