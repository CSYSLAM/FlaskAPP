# 手机 WAP 文字页设计 Skill

## 目标
为本项目所有界面提供统一的设计准则：以“彩色文本层级 + 极轻量交互 + 极低资源占用”为核心，做成老派但清晰、耐用、移动端友好的文字游戏界面。

这个 skill 适用于：
- 游戏主场景
- 战斗结果
- 地图与传送
- 背包、角色、技能、商店
- 活动、剧情、任务、排行榜

## 设计哲学

### 1. 文字是主角
页面首先是“可读”，其次才是“好看”。

用户进入页面后，应当在 1 到 2 秒内看懂三件事：
- 我现在在哪
- 我拿到了什么 / 正在面对什么
- 我下一步该点哪里

### 2. 颜色承担信息分层
WAP 文字页不靠厚重组件分层，而靠文本颜色做语义区分。

推荐层级：
- 默认正文：黑色或接近黑色
- 链接操作：蓝色 `#136ec2`
- 危险/失败/扣血：红色系
- 收益/恢复/正反馈：绿色系
- 稀有/史诗/传奇：用品质色或少量炫彩字
- 地点、身份、称号、活动名：可以比正文更亮，但不能满屏抢戏

### 3. 炫彩要少而准
渐变炫彩文字是这个风格里的“奖励感放大器”，不是默认文本样式。

只建议用于：
- 稀有掉落
- 特殊称号
- BOSS 名
- 大事件提示
- 副本结算标题

不建议用于：
- 普通描述
- 所有按钮
- 整页大段文本
- 导航区

经验法则：
- 单屏 1 到 2 个炫彩点足够
- 连续三行以上炫彩会破坏可读性
- 如果一个页面已经有很多红、绿、蓝信息色，就减少炫彩使用

## 页面结构模板

典型结构从上到下建议是：

1. 功能导航
2. 当前场景/标题
3. 状态信息
4. 主体内容
5. 当前可执行动作
6. 返回或继续入口
7. 报时/尾部轻信息

副本任务链推荐额外拆成：
- NPC 任务入口页
- 任务详情页
- 接受任务确认页
- 回 NPC 交任务页
- 完成任务确认页
- 场景领奖页

推荐例子：

```html
<a href="#">刷新</a> .<a href="#">角色</a> .<a href="#">背包</a> .<a href="#">商城</a><br/>
*豫州(黄巾之乱)<br/>
体力: 20/30<br/>
你发现了黄巾残党。<br/>
<span class="rarity-rare">获得稀有战利品线索</span><br/>
<a href="#">前往战斗</a><br/>
<a href="#">返回城镇</a><br/>
小Q报时(17:40)<br/>
```

## 交互原则

### 1. 链接优先
能用文本链接完成的，就不要做厚按钮。

适用场景：
- 前进 / 返回
- 接任务
- 使用道具
- 查看详情
- 打开地图
- 领取奖励

### 2. 交互反馈必须轻
用户点击后，要立即知道页面“收到了输入”，但反馈必须便宜。

推荐做法：
- 全页 loading 遮罩
- 简单显隐
- 少量浮层提示
- 局部颜色高亮

不推荐：
- 长时间动画
- 多段复杂过渡
- 大型弹窗流程
- 前端状态机式交互

### 3. 一屏一个主动作
如果一个页面同时出现十几个动作，用户会失去节奏。

建议：
- 主动作直接露出
- 次动作收敛到下方
- 低频动作放返回页或详情页

副本页额外规则：
- 任务详情页不要提前显示 `完成任务`
- 达成目标后，先通过 `传送` 回任务 NPC
- 只有 NPC 下钻后的交任务页才出现 `完成任务`
- 最终阶段完成后，先显示任务完成确认页，再回场景显示 `【 领取通关奖励 】`

## 资源约束

### 必须低资源
- 单页尽量少量内联 CSS/JS
- 不引入前端框架
- 不依赖大图背景
- 不加载视频、复杂图表、轮播
- 不使用重阴影、毛玻璃、长关键帧动画

### 兼容低端设备
- 单栏布局
- 基础字号 `16px`
- 点击目标靠文字和间距，不靠复杂组件
- 不假设高刷新率和强性能

## 视觉语法

### 基础约定
- `body`：左右留白 10px
- 默认字号：`16px`
- 行高：`1.5` 到 `1.7`
- 链接色：`#136ec2`
- 链接无下划线
- 主要内容左对齐

### 导航写法
使用点分隔的纯文字导航：

```html
<a href="#">角色</a> .<a href="#">背包</a> .<a href="#">聊天</a> .<a href="#">活动</a>
```

### 奖励写法
奖励信息要短、直、可扫读：

```html
经验+1500<br/>
银两+150<br/>
获得：止血草 x2<br/>
```

### 状态写法
状态应先结论后细节：

```html
你受了轻伤。<br/>
生命-32<br/>
```

而不是写成很长的段落。

### 副本任务页写法
推荐固定结构：

```html
副·大破黄巾一<br/>
任务来自:灵帝 传送<br/>
前往「豫州」击杀5名『黄巾士兵』<br/>
快速前往目的地<br/>
任务奖励:<br/>
经验+1500<br/>
银两+150<br/>
任务进度:<br/>
已击败黄巾士兵(6级)(3/5)<br/>
放弃任务<br/>
返回<br/>
返回游戏<br/>
```

如果阶段目标有多个首领，进度应逐行展示，不要压缩成单一总数：

```html
已击败【精】张宝(10级)(0/1)<br/>
已击败【精】张梁(10级)(0/1)<br/>
已击败【精】张角(10级)(0/1)<br/>
```

## 通用组件建议

### 1. Loading
所有可跳转页建议保留统一 loading：

```html
<div id="loading"><span></span></div>
<script>
var aLinks=document.getElementsByTagName("a");
for(var i=0;i<aLinks.length;i++){aLinks[i].onclick=function(){document.getElementById('loading').style.display='block';};}
var inputs=document.getElementsByTagName("input");
for(var j=0;j<inputs.length;j++){if(inputs[j].type==='submit'){inputs[j].onclick=function(){document.getElementById('loading').style.display='block';};}}
setTimeout(function(){document.getElementById('loading').style.display='none';},350);
</script>
```

### 2. 地图
地图应保持字符块感和轻量感，不要变成大面积图形 UI。

要求：
- 小尺寸 block
- 当前位置突出
- 相邻场景用米黄色或浅色块
- 地图可一键开关

### 3. 弹层
只用于短信息或道具详情。

要求：
- 内容短
- 关闭路径简单
- 不承载复杂流程

## 设计时的 Do / Don't

### Do
- 优先让页面“扫一眼就懂”
- 用颜色表达优先级
- 用 `<br/>` 和短句控制节奏
- 保持页面像短信/公告/战报一样直接
- 让重要操作永远出现在正文附近

### Don't
- 不要把页面做成现代后台风卡片墙
- 不要堆大按钮、大圆角、大片留白
- 不要引入多层组件嵌套
- 不要让颜色只为装饰服务
- 不要为了“精致”牺牲低端机可打开性

## 页面验收清单

做完一个页面后，用下面的问题自检：

- 这是文字游戏页，还是被做成了普通 H5 活动页？
- 用户能不能在首屏直接找到主动作？
- 颜色是不是在表达语义，不是在乱装饰？
- 有没有可以删掉的边框、卡片、阴影、动画？
- 去掉 70% 样式后，这页还能不能清楚使用？
- 这页是否延续了项目已有的 WAP 气质？

从现在开始，`templates/` 下所有新界面和改版界面都必须参考本 skill。

黄巾起义副本的强约束：
- 阶段节奏以 `agent_context/huangjin-trial-live-notes.md` 为准
- 最终决战详情页应采用三行首领进度

- 通关后先回场景，再显示 `【 领取通关奖励 】`

如果某个页面需要例外，必须同时满足：
- 明确有玩法或信息结构上的必要
- 不破坏移动端文字优先原则
- 不显著增加资源开销
- 仍然能和现有页面看起来属于同一个游戏
